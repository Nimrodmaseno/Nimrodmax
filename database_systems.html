CIR106: Database Systems Complete Notes

1. Definition of Database Systems

A Database System (DBS) is a software system that allows for the efficient storage, retrieval, manipulation, and management of structured data.

Components:

  • Database (DB): Organized collection of data
  • Database Management System (DBMS): Software that manages the database
  • Applications: Programs that interact with the database
  • Users: People who interact with the system

2. Types of DBMS

Type Description Examples
Relational (RDBMS) Organizes data in tables with rows and columns MySQL, Oracle, SQL Server
Hierarchical Tree-like structure with parent-child relationships IBM IMS
Network Graph structure allowing complex relationships IDMS
Object-Oriented Stores data as objects with attributes and methods MongoDB, ObjectDB
NoSQL Handles unstructured/semi-structured data Cassandra, Redis, Neo4j

3. Advantages and Disadvantages

Advantages Disadvantages
  • Data sharing and integration
  • Reduced redundancy
  • Improved data security
  • High setup cost
  • Complexity in design
  • Performance overhead

4. Major Parts of Database System

  1. Data: The actual information stored
  2. Hardware: Physical storage devices
  3. Software: DBMS and related programs
  4. Users: End-users, programmers, DBAs
  5. Procedures: Rules for database usage

5. Functions of Database Administrator

  • Database design and implementation
  • User management and access control
  • Backup and recovery procedures
  • Performance monitoring and tuning
  • Security enforcement

6. Database Architecture and Environment

Levels of Abstraction:

  1. External Level: User-specific views
  2. Conceptual Level: Logical structure
  3. Internal Level: Physical storage details

Data Independence:

  • Logical: Changes in conceptual schema don't affect external views
  • Physical: Changes in storage don't affect logical schema

7. Components of DBMS

Component Function
DDL (Data Definition Language) Defines database structure (CREATE, ALTER, DROP)
DML (Data Manipulation Language) Manages data (INSERT, UPDATE, DELETE)
DCL (Data Control Language) Controls access (GRANT, REVOKE)
Query Processor Executes and optimizes queries

8. Features of Report Generator

  • Customizable layouts
  • Data filtering and sorting
  • Graphical representations
  • Export options (PDF, Excel)
  • Scheduling capabilities

9. Typical DBMS Functions

  • Data storage and retrieval
  • Transaction management
  • Concurrency control
  • Security and authorization
  • Backup and recovery

10. Major Components of DBMS

  1. Storage Manager
  2. Query Processor
  3. Transaction Manager
  4. Recovery Manager
  5. Buffer Manager

11. Database Life Cycle

  1. Requirement Analysis: Identify needs
  2. Design: Conceptual, logical, physical
  3. Implementation: Create and populate
  4. Testing: Validate performance
  5. Maintenance: Optimize and update

12. Types of Data Models

  • ER Model: Entity-Relationship diagrams
  • Object-Oriented: Objects with attributes/methods
  • Binary: Relationships between two entities

13. ER Model Components

Rectangles Entities (e.g., Student, Course)
Ellipses Attributes (e.g., Student_ID, Name)
Diamonds Relationships (e.g., "Enrolls in")

Types of Relationships:

  • One-to-One (1:1)
  • One-to-Many (1:N)
  • Many-to-Many (M:N)

14. Data Normalization

Normal Forms:

Form Description
1NF No repeating groups
2NF No partial dependencies
3NF No transitive dependencies

Advantages:

  • Reduces redundancy
  • Improves data integrity

Disadvantages:

  • Increased complexity
  • Slower queries

15. Relational Database System

Relational Data Structure:

  • Tables (relations)
  • Rows (tuples)
  • Columns (attributes)

Properties of Relations:

  • No duplicate rows
  • Order doesn't matter
  • Atomic values

16. SQL

Basic Structure:

  • SELECT: Retrieve data
  • INSERT: Add new records
  • UPDATE: Modify data
  • DELETE: Remove records

17. Transaction Management

ACID Properties:

  • Atomicity: All or nothing
  • Consistency: Valid state transitions
  • Isolation: Independent execution
  • Durability: Permanent changes

Concurrency Problems:

  • Lost update
  • Dirty read
  • Non-repeatable read

Control Techniques:

  • Locking methods
  • Timestamp ordering
  • Optimistic concurrency
